# STANDARDS

standars/checklist

- clean code
- snake_case for function names, ALL_CAPS for types and consts
- clear names
- `// ============================================` comments at the top after import staments before code, and at the bottom of file.
- can have ```// funtion_name``` comment just before a funtion, other than that not many comments are necessary.
- run `bun run tsc --noEmit` and `bunx prettier . --write`, to verify and perttify code
- console logs can have ```("=============================");``` above and below what was consol logged see ```src/ts/hello.ts``` for example
- everthing should be spelled right, so make sure to do spell check.

if you see code here that does not follow thse standards fix it.
